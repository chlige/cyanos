Interface Concepts and Use

The web based interface is the primary tool for the management of data in the CYANOS database. This interface was designed to work with W3C compliant web browsers, e.g. Mozilla Firefox, Opera and Apple Safari. No additional software is needed to access the web interface.

Web Interface

Data in the CYANOS database can be managed through the use of webpage forms or through the upload of Microsoft Excel XML or OpenOffice.org spreadsheets. In addition, data files can be linked to samples, separations and bioassays. For these data files, any binary or text format is acceptable thus, one could link the proprietary raw data, e.g. a Bruker FID or Agilent Chemstation datafile, or a standard representation, such as a PDF report or PNG/BMP graphic.

Menu Bar

All pages in the CYANOS web interface have a menu bar at the top to allow quick access to the different pages within the application.

The available menus and sub-menus will depend on the access rights of the user, so it is possible that certain menu items may be missing due to access restrictions.

Project Information

The CYANOS database has the ability to group resources by project. Projects can be created in the Project Manager interface, which is accessed from the menu Cyanos > Manage Projects. From this interface, one can create new projects or modify the description of existing projects.

Spreadsheet Upload

Some tasks, e.g. adding bioassay data, require the entry of large amounts of data. To facilitate these tasks, a spreadsheet upload feature was created. The upload functions of the CYANOS database allow one to upload a spreadsheet to the web interface then have the interface parse the data and update the database accordingly. Currently the allowed spreadsheet formats are:

  • Microsoft Excel XML (Office version 2003 or higher)

  • OpenOffice.org (version 2.0 or higher)

Currently the following upload modules are included with the CYANOS web interface. All upload modules, including custom modules, can be accessed from the upload page via the menu path Cyanos > Upload Data.

In addition to these modules, it is possible to add custom upload modules to the CYANOS web interface. Details on developing and installing custom upload modules can be found in the Developer's Guide.

User Accounts

User accounts can be created for each user that will access CYANOS. Administrators can add, modify, or delete users via the User Administration page (Cyanos >Manage Users). All users have a defined user ID and password. In addition, the full name and email address of the user can be stored. Note: CYANOS utilizes email for password resets as well as the initial setup of an user account.

User accounts will also have a list of associated roles and permissions, which can be either global or project specific. The roles defined in CYANOS are "admin", "culture", "sample", "assay", and "project". For each role, the permissions of read (R), write (W), create (C), and delete (D) can be assigned to further refine access rights.

RoleObjects
adminUser accounts, CYANOS configuration
cultureStrains, Field Collections, Isolations, Inoculations, Cryopreservations, Harvests
sampleSamples, Sample Collections, Separations, Compounds
assayBioassays
projectProjects
The roles and permissions in CYANOS are additive, that is project specific roles and permissions add to, rather than substitute, the user's set of global roles and permissions. For objects that are not assigned to a project, the "Null Project" roles and permissions will be used in combination with the global roles to determine access.

For example, a user has been assigned the roles listed in the following table.
RoleGlobalProject AProject B
cultureR,WR,W,C,D
sampleRW
assayR,WR

For objects part of Project A, the user would have the roles culture(RWCD), sample(R), and bioassay(RW). In Project B, the user would have the roles culture(RW), sample(RW), and bioassay(RW). Throughout this guide, required permissions for various CYANOS actions will be listed to help administrators properly assign roles.
